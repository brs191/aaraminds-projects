# Ticket V4-07-Go — Go engine: key findings by ARM resource id (id ‖ name)

**Type:** bug / parity · **Effort:** M (~half day) · **Needs:** Go 1.25 toolchain (CI or local)
**Blocks:** twin-drift parity on multi-subscription estates · **Status:** READY

## Context

The Python reference engine was fixed (2026-06-15) to key findings by `rid()` = ARM `id` when present,
else bare name — because bare names are not unique across subscriptions and two same-named NICs were
merging *at input* (`nics = {name: nic}` dropped one). See `engine/reference/analyze.py` (`rid`, `eff_for`)
and `engine/reference/test_resource_id.py`. The Go production engine still keys by bare `Name`
(adversarial review HIGH-3) and so diverges from the corrected twin. This ticket brings Go to parity.

The change is **additive**: fixtures without `id` give `rid == name`, so all golden tests are unchanged.

## Changes (exact)

### 1. `internal/graph/model.go` — add `ID` to the resource structs whose findings key by name
```go
type NIC struct {
    Name string `json:"name"`
    ID   string `json:"id,omitempty"` // ARM resource id; identity key across subscriptions (V4-07)
    // ... existing fields ...
}
type PublicIP struct {
    Name string `json:"name"`
    ID   string `json:"id,omitempty"` // V4-07
    // ... existing fields ...
}
```
(Follow-up, optional: same for `PrivateEndpoint`, gateways, AKS — any resource the engine keys a finding
on by name. NIC + PublicIP cover the v1 finding families.)

### 2. `internal/analyze/analyze.go` — key by id, look up NW tables by id with name fallback
Add a helper:
```go
func rid(name, id string) string { if id != "" { return id }; return name }
```
In `Analyze()` build the NIC map by identity (was keyed by `n.Name`):
```go
nics := map[string]graph.NIC{}
for _, n := range rg.NetworkInterfaces { nics[rid(n.Name, n.ID)] = n }
```
Effective-rules / routes lookups (NW tables are name-keyed in current fixtures, id-keyed in a real
multi-sub estate) — mirror the Python `eff_for`:
```go
rules := nw.EffectiveSecurityRules[rid(nic.Name, nic.ID)]
if rules == nil { rules = nw.EffectiveSecurityRules[nic.Name] }
routes := nw.EffectiveRoutes[rid(nic.Name, nic.ID)]
if routes == nil { routes = nw.EffectiveRoutes[nic.Name] }
```
Set `Finding.Resource = rid(nic.Name, nic.ID)` for the NIC finding families (over-permissive
reachable/latent, firewall DNAT, missing segmentation) and `rid(pip.Name, pip.ID)` for orphaned PIP.

### 3. `adapter/resourcegraph.go` — project + populate `id`
- `kqlNICs` (line ~375): add `id,` to the `| project` list (it currently omits it).
- `kqlPublicIPs`: add `id,` to its `| project` list.
- `parseNICs` (~822, `nic := graph.NIC{...}`): set `ID: getString(row, "id")`.
- `parsePublicIPs` (~759): set `ID: getString(row, "id")`.

### 4. `renderer/drawio.go` — old renderer keys by name (lines 167, 293, 334)
Key `nicFindings` by `rid(nic.Name, nic.ID)` and index it the same way, OR note this renderer is
superseded by the Phase-4 Python pipeline (`phase-4/viz/`) and leave it with a `// TODO V4-07` if you are
retiring it. Do not leave it keyed by `nic.Name` while findings are keyed by id (silent mismatch).

### 5. New test `internal/analyze/resourceid_test.go`
Mirror `engine/reference/test_resource_id.py`: two NICs named `nic-web` with distinct `ID`s in two
subscriptions, id-keyed effective rules → assert two distinct findings (`/subs/subA/...` Critical,
`/subs/subB/...` High), and assert deterministic `(Resource, Type, Evidence)` order.
