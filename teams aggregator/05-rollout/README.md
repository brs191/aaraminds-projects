# 05 · Rollout

Go-to-market, communications, training, and launch operations.

## Suggested layout (create as needed)

- `plan/` — phased rollout plan (mirrors PRD §14: alpha → beta → GA → integrations)
- `comms/` — announcement emails, Teams posts, leadership briefings, FAQ
- `training/` — admin guide, project-lead quick-start, recorded walkthroughs
- `feedback/` — survey templates, NPS structure, channels for beta feedback
- `metrics/` — success-metric tracking (see PRD §12 targets), weekly dashboards

## Pre-launch checklist (from PRD §10 + §14)

- DPIA / privacy review with IT and Security — **pre-launch gate**
- Teams app catalog approval — admin path confirmed and submitted
- Reserved capacity arranged with the AskAT&T team for the < 30s SLO
- Labeled ground-truth set of 200 Aara messages prepared for recall measurement
- Sentinel runbook (`Aara-IR-v1`) wired into the SOC playbook
- Admin training session scheduled for the BCLM admin (project lead)
- Exec-stakeholder list confirmed for the scheduled weekly digest
